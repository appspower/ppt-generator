# Phase A4 — 사용자 자산 캡처 가이드 (PwC cover/closing/divider)

> 목적: Step 4 옵션 B의 사용자 작업 부분
> 작업: 회사PC에서 PwC 슬라이드 5~10장을 추가 캡처하여 마스터 풀에 보강
> 예상 효과: +5~10점 (cover/closing/divider 결손 해결)

---

## 1. 왜 필요한가

5 시나리오 시각 검증 결과, 다음 슬라이드들이 **마스터 풀에 빈약**해서 자동 생성 시 어색함:

| role | 마스터 풀 갯수 | 문제 |
|---|---|---|
| **closing** | 1 | 5 시나리오 모두 같은 결론 슬라이드 사용 |
| **situation** | 1 | 단일 슬라이드만 사용 가능 |
| **risk** | 1 | 단일 슬라이드 |
| **complication** | 2 | 2 시나리오 이상 동시 사용 어려움 |
| **opening** | 8 | 거의 빈 페이지 디자인뿐 |
| **benefit** | 3 | 빈약 |
| **divider** | 9 | 일부만 본격 디자인 |

→ 이 7개 role에 대해 **PwC 실 슬라이드 5~10장 추가 캡처** 필요.

---

## 2. 어떤 슬라이드를 캡처해야 하는가

회사PC에서 PwC 본사/계열 컨설팅 자료 PPT를 열어 **다음 디자인 패턴** 슬라이드를 캡처하세요.

### 우선순위 1 (필수, 5장)
1. **본격 cover 슬라이드 1장**
   - 큰 타이틀 + 부타이틀 + 발표자/일자 + 회사 로고/색상 띠
   - 빈 페이지가 아닌, 디자인 요소가 풍부한 cover

2. **closing/conclusion 슬라이드 1장**
   - "Key Takeaways" 또는 "Recommendation Summary" 식의 결론 페이지
   - 3~5개 핵심 메시지를 박스/카드로 정리

3. **section divider 슬라이드 1장**
   - "Section 2. 분석 결과" 식의 큰 섹션 구분
   - 큰 숫자/제목 + 서브 메시지

4. **situation/현황 진단 슬라이드 1장**
   - "현재 상황" 식의 As-Is 분석
   - 2~4 핵심 facts를 매트릭스/카드로 표시

5. **risk/위험 평가 슬라이드 1장**
   - 위험 매트릭스 (probability × impact 2x2)
   - 또는 위험 항목 카드 3~5개

### 우선순위 2 (선택, 5장 추가)
6. opening 1장 — 표지보다 1단계 안쪽 page (Agenda 직후)
7. complication 1장 — 문제 정의 페이지
8. benefit 1장 — 기대 효과 차트/매트릭스
9. opening 1장 — 다른 스타일의 cover
10. divider 1장 — 다른 스타일의 섹션 구분

### 캡처 기준
- ✅ **PwC 톤 (전문적, 깔끔, 색상 팔레트 통일)**
- ✅ 컨텐츠 자리에 `~~` 또는 placeholder 텍스트가 있어도 OK (어차피 우리가 채움)
- ✅ 도형/색상/폰트가 풍부할수록 좋음
- ❌ 사진 위주 / 단순 텍스트만 / 너무 빈 페이지

## 3. 캡처 방법

### 방법 A — PowerPoint에서 직접 슬라이드 복사 (권장)
1. 회사PC에서 원본 PwC PPT 열기
2. 해당 슬라이드 **선택 → 복사** (Ctrl+C)
3. 새 PPT 만들기 → 붙여넣기 (Ctrl+V) — 디자인 100% 보존
4. 5~10장 모은 새 PPT를 `pwc_extras.pptx`로 저장

### 방법 B — Slide Library 활용 (있으면)
- PwC Knowledge Navigator 등에서 "cover", "closing", "section divider" 검색
- 대표적인 5~10장 export

### 방법 C — Snip 캡처는 불가
- 우리는 도형/텍스트 정보가 필요 — 단순 PNG 캡처는 안 됨
- 반드시 .pptx 파일로 가지고 와야 함

## 5. 캡처 후 자동 작업 (Claude가 수행)

사용자가 `pwc_extras.pptx` 저장하면:
1. **자동 추가 라벨링**: 새 슬라이드 N장 → 기존 catalog에 추가
2. **자동 narrative_role 라벨링**: cover/closing 등 자동 분류
3. **paragraph 추출 + role 라벨링**: 기존 파이프라인 재사용
4. **5 시나리오 재실행**: 새 자산 활용으로 점수 측정
5. **결과 보고**: 기대 +5~10점

→ 사용자 시간: 캡처 작업만 (1~2시간 추정)
